# v1.20.1 — Standings & Dashboard Resilience Hotfix

## Purpose

v1.20.1 is a focused corrective release for a provider-data edge case: a completed fixture without an explicit group label caused group standings sorting to compare an absent group value with normal group text.

The release also ensures that a failure in a secondary dashboard panel cannot hide otherwise healthy fixture and Matchday data.

## Fixed

### Safe group standings

- `build_group_standings()` now excludes completed fixtures that do not have an explicit `group_name`.
- Ungrouped fixtures remain available in the fixture browser and match-detail views; the release only keeps them out of group rankings where no group can be stated factually.
- `/standings` and the dependent `/ai/insights` route remain available when grouped and ungrouped completed fixtures coexist.

### Independent dashboard panels

- Dashboard core data requests now use independent settlement rather than one all-or-nothing load.
- A standings, group-insights, or structured-AI-insights failure now produces a scoped panel message while healthy fixture and Matchday data continue to render.
- Initial load and filter application share the same independent-loading behavior.

### Static assets

- Dashboard and Live Match Centre CSS and JavaScript cache-busting values advance to `v1.20.1`.

## Preserved boundaries

This release does not:

- alter fixture records, provider responses, provider schedules, fixed daily sync times, timezone, freshness thresholds, or stored event data;
- trigger provider syncs, database writes, Telegram delivery, or browser polling;
- change Docker, Cloudflared, Ollama, Windows runtime configuration, or an active runtime `.env`;
- infer a group for an ungrouped fixture or place it into a group ranking.
