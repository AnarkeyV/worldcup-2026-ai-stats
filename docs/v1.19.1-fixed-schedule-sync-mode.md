# v1.19.1 — Fixed Schedule Sync Mode Truthfulness

## Purpose

v1.19.1 is a small dashboard-only truthfulness patch for the Provider Sync Runtime card.

The active scheduler is configured as fixed daily Singapore-time slots:

```text
03:45 · 09:45 · 12:45 Asia/Singapore
```

Earlier dashboard wording displayed **Every 30 min** because `interval_minutes` remains in the status payload for backward compatibility. That field is not the active runtime schedule when `mode` is `fixed_daily_times`.

## Changed

- The dashboard now checks `scheduler.mode` first.
- When `mode` is `fixed_daily_times`, it renders the configured daily slots and timezone:

```text
Fixed daily: 03:45 · 09:45 · 12:45 (Singapore time)
```

- Interval wording remains available only for genuine interval-mode schedules.
- The dashboard JavaScript cache-buster advances to `v1.19.1`.

## Preserved boundaries

This release does not:

- change fixed schedule slots, scheduler enablement, timezone, or freshness thresholds;
- create a provider request, sync, write, or backfill;
- change fixture, score, event, or status data;
- change Telegram, Docker, Cloudflared, Ollama, database data, or an active Windows `.env`;
- infer that a fixture is live from kickoff time.

The v1.19.0 Freshness Context contract and v1.18.1 conservative unknown-fixture display contract remain unchanged.
