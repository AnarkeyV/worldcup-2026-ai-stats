# v1.22.1 — Matchday Changes Load-Order Hotfix

## Summary

v1.22.1 is a source-only hotfix for the browser-local **What changed since your last visit** panel. It fixes a load-order issue where an early Matchday render could cache a temporary empty fixture list before the normal stored fixture request completed.

## Fixed lifecycle

- The panel reports a loading state while the normal fixture list is still being obtained.
- A temporary empty in-memory list cannot be saved or reused as a true no-knockout-fixtures comparison result.
- After a successful stored fixture load, the comparison cache is reset and the existing browser-local baseline can correctly report first visit, changes, or unchanged state.
- After a failed fixture load, the panel renders a distinct unavailable state without reading, replacing, or discarding an existing browser-local baseline.
- A genuine completed fixture load with no supported stored knockout fixtures remains distinct from loading and unavailable data.

## Preserved boundaries

- The browser-local panel compares only already loaded stored fixture data. It does not call the provider, trigger sync, send Telegram, write the database, poll, or start a background job.
- Provider-backed fixtures remain the source of truth. Road to the Final does not infer advancement, winners, pairings, penalties, extra time, aggregate results, or live state.
- Provider schedule, freshness thresholds, Cloudflared, Ollama, Docker configuration, database data, active Windows `.env`, Official Match Video policy, and Windows host configuration remain unchanged.
