# v1.17.0 — Provider-Backed Match Story and Official Watch

## Release Summary

v1.17.0 improves the match-detail experience by turning locally stored provider data into a compact, factual match story. It also establishes a strict outbound-only policy for official match-video destinations.

The release is intentionally conservative: it prefers an unavailable or partial state over a polished but unsupported chart, event, or video claim.

## What It Adds

### Read-only match story

`GET /fixtures/{fixture_id}/story` returns a computed response built only from local records:

```text
Fixture
MatchDetail
OfficialMatchVideo
```

The route does not call a provider, write data, trigger sync, backfill match detail, send Telegram, scrape a website, or change scheduler state.

### Score progression with reconciliation

A score-progression rail appears only if:

- fixture home and away scores exist;
- every stored goal has a valid side and usable minute; and
- stored goal counts match the stored final score exactly.

If the data fails that check, the response exposes an unavailable state and reason rather than inventing a sequence.

### Data-backed match events and statistics

The Story view can show:

- goals
- cards
- substitutions
- valid stoppage-time events
- paired comparisons for provider-supplied possession, shots, shots on target, expected goals, accurate passes, corners, and fouls

A statistic is omitted unless both teams have a valid stored provider value. Missing values are not converted into zeroes or 50:50 comparison bars.

### Official Watch policy

The initial policy accepts only manually verified, match-specific HTTPS links from:

- approved FIFA web hosts
- exact FIFA YouTube video URLs
- approved meWATCH hosts

Every accepted card is outbound only and carries safe external-link behaviour:

```text
target="_blank"
rel="noopener noreferrer"
```

No video is embedded, downloaded, rehosted, proxied, scraped, or discovered automatically.

Until a verified match-specific link exists in the local registry, the dashboard reports that clearly and offers labelled FIFA/meWATCH coverage-hub fallbacks. Availability can be delayed or territory-dependent.

## Dashboard Changes

The existing match-detail panel now leads with **Story**, followed by:

```text
Timeline
Stats
Lineups
```

The Story tab combines provider provenance, a safe score progression, key events, compatible team statistics, and Official Watch state/cards. The layout remains mobile-first and uses explicit unavailable and partial states.

Dashboard asset URLs now use the `v1.17.0` cache-bust suffix.

## Data and Runtime Boundaries

- No provider sync or data backfill occurs when a story is read.
- No scheduled sync, Telegram setting, or manual completed-match alert setting changes.
- No active `.env` file is modified.
- No video source is hard-coded as a match-specific highlight.
- The new `official_match_videos` table remains empty until a future controlled curator workflow adds individually verified records.
- The existing ORM bootstrap can create that empty table on a later approved runtime deployment; no runtime deployment is performed by source release preparation.

## Future Follow-up

The natural follow-up is a controlled, operator-run official-link curator import workflow with a dry-run manifest, policy validation, idempotent upsert, and audit-friendly rejection output. It should not be replaced by scraping or a public browser-write form.
