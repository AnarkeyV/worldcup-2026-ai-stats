# v1.22.0 — Confirmed Knockout Path & Matchday Changes

## Summary

v1.22.0 is a source-only live-product UX release for the knockout phase. It adds a compact Road to the Final based solely on stored provider-backed fixtures and a browser-local summary of confirmed knockout changes since the visitor's previous successful dashboard load.

## Confirmed Knockout Path

- The supported display order is Round of 32, Round of 16, Quarter-finals, Semi-finals, Third-place Playoff, then Final.
- A stage appears only when matching stored provider-backed fixtures exist.
- There are no empty future brackets, placeholder slots, assumed winners, inferred pairings, or fabricated advancement paths.
- Existing fixture cards retain the established conservative fixture-state presentation.
- Selecting a path fixture preserves the existing fixture browser and selected-match detail, including Match Story, timeline, statistics, lineups, AI summary, player leaders, freshness, and Official Watch states.
- A completed draw never implies an advancing team. The dashboard does not infer penalties, extra time, aggregate results, winners, or progression.

## Matchday Changes Since Last Visit

- The dashboard compares only already loaded supported knockout fixtures against a small versioned `localStorage` snapshot saved by the same browser.
- The snapshot uses a stable provider external identifier when available, with stored fixture identifier fallback, along with canonical stage, raw status, teams, and scores.
- The summary distinguishes first visit, newly stored fixture, newly completed fixture, confirmed score/status change, unchanged state, and unavailable local comparison state.
- The browser-local summary does not call a provider, trigger sync, send Telegram, write the database, poll, or start a background job.
- It remains separate from provider-snapshot freshness, local stored match-detail coverage, and Live Match Centre sync-change evidence.

## Accessibility and mobile behavior

- Stage controls retain semantic buttons, visible focus styles, keyboard access, and 44px minimum interactive targets.
- The experience uses compact stage browsing rather than a decorative, cramped full-width bracket.
- Group Stage progressive disclosure remains in place; group-era content does not return to the default Matchday hierarchy.
