# v1.20.0 — Matchday Home & Compact Sync UX

## Purpose

v1.20.0 makes the dashboard easier to use during a busy matchday without overstating the freshness, certainty, or source of stored tournament data.

The release creates a clearer **Now / Next / Latest** Matchday hierarchy, keeps provider-snapshot freshness distinct from local match-detail coverage, and reduces navigation and Sync-panel density on mobile.

## Changed

### Matchday Home

- Matchday cards now prioritise **Now**, **Next**, and **Latest**.
- A compact **Data trust** strip presents stored provider-snapshot freshness and links directly to Sync details.
- **Match detail coverage** remains separately labelled, so local stored-detail coverage is not confused with provider-snapshot freshness.
- A fixture explicitly stored as live is labelled:
  - **Live in latest stored refresh** when the stored snapshot is fresh or aging;
  - **Last recorded live** when the stored snapshot is stale or otherwise not current enough to support present-tense wording.
- A future fixture derived from a valid stored kickoff is labelled **Upcoming from stored kickoff** and states that provider match status is unavailable.

### Fixture-state consistency

- Matchday hero selection now reuses the dashboard's established conservative display-state classifier.
- Explicit stored live, completed, and scheduled states remain authoritative.
- A future `unknown` fixture is only treated as upcoming when the existing stored-kickoff rules are satisfied.
- A past or unresolved `unknown` fixture remains unavailable. Time is never used to infer that a match is live.

### Navigation, accessibility, and Sync

- Desktop and mobile primary navigation prioritise **Matchday**, **Matches**, **Groups**, and **Players**.
- **Overview**, **Data**, **Insights**, and **Sync** remain reachable through an accessible **More** disclosure.
- Matchday cards, fixture status/group controls, navigation, and disclosure controls use 44px minimum tap targets and visible keyboard focus styling.
- Fixture status/group controls retain the existing explicit categories: **Live**, **Completed**, **Upcoming**, and **Data unavailable**.
- Sync's **What changed?** section shows its factual summary first and places individual change records behind native **Show change details** disclosure.

### Static assets

- Dashboard and Live Match Centre CSS and JavaScript cache-busting values advance to `v1.20.0`.

## Preserved boundaries

This release does not:

- change provider schedules, fixed daily slots, timezone, freshness thresholds, provider requests, sync history, or stored fixture/event data;
- trigger provider syncs, database writes, Telegram delivery, or browser polling;
- change Docker, Cloudflared, Ollama, Windows runtime configuration, or an active runtime `.env`;
- claim that a stored live fixture is current when snapshot freshness is stale or unavailable;
- turn a future stored kickoff into a provider-confirmed scheduled status.

The v1.19.1 fixed-schedule display contract, v1.19.0 Freshness Context contract, and v1.18.1 conservative unknown-fixture display contract remain intact.
