# v1.8.0 — AI Insights Upgrade

## Purpose

This guide documents the **v1.8.0 — AI Insights Upgrade** milestone for the World Cup 2026 AI Stats project.

The milestone improves the project’s AI story by adding structured, dashboard-ready insights generated from:

- fixture data
- completed results
- standings context
- group and team filters
- provider sync runtime status

The goal is to make the AI layer more useful during a portfolio review, interview, or live technical walkthrough while keeping the demo reliable when Ollama or a local LLM is unavailable.

## What v1.8.0 Adds

### Structured AI Insights Endpoint

A new endpoint is available:

```text
GET /ai/insights
```

It returns structured insight cards instead of one loose paragraph.

The endpoint supports:

```text
GET /ai/insights
GET /ai/insights?group_name=Group%20A
GET /ai/insights?team=Mexico
GET /ai/insights?team=MEX
GET /ai/insights?limit=3
```

Example response shape:

```json
{
  "status": "ok",
  "mode": "fallback",
  "provider": "rules_based_ai_insights",
  "model": "rules_based_v1",
  "summary": "4 fixtures are available for AI insights, with 4 completed fixture data. The latest provider sync completed successfully.",
  "filters": {
    "group_name": null,
    "team": null,
    "limit": 5
  },
  "metadata": {
    "fixture_count": 4,
    "completed_count": 4,
    "live_count": 0,
    "scheduled_count": 0,
    "other_status_count": 0,
    "teams_analyzed": 8,
    "groups_analyzed": 4,
    "sync_status": "success"
  },
  "insights": [
    {
      "title": "Fixture data available",
      "category": "fixtures",
      "message": "4 fixtures are available. Completed: 4. Live: 0. Scheduled: 0."
    }
  ]
}
```

## Backend Service

The milestone adds:

```text
backend/app/services/ai_insights_service.py
```

The service generates deterministic, fallback-safe insight output.

It can identify:

- fixture availability
- completed results
- group leaders
- strongest attacks
- provider sync runtime status

This is intentionally not a betting or prediction engine. It explains the current state of the app using data already available inside the system.

## Why This Is Demo-Safe

The existing local Llama/Ollama workflow remains available through:

```text
GET /ai/health
GET /ai/fixtures/summary
GET /ai/fixtures/{fixture_id}/summary
```

The new endpoint:

```text
GET /ai/insights
```

does not require Ollama.

This means:

- the dashboard can still show AI-style insights when Ollama is offline
- the Windows Docker/runtime demo remains reliable
- the feature is fully testable through deterministic rules
- reviewers can see AI-adjacent behavior without waiting for local model generation

- It does not require Ollama.
- Existing local Llama summary endpoints remain available.
- Runtime sync status is intentionally in-memory.
- API-Football credentials remain optional and environment-driven.
- No secrets should be committed to the repository.
