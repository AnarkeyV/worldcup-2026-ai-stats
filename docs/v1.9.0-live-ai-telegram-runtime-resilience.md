# v1.9.0 — Live AI, Telegram Mobile Alerts & Windows Runtime Resilience

## Summary

v1.9.0 turns the World Cup 2026 AI Stats project from a local/demo workflow into a personally usable live dashboard.

This release focuses on four major improvements:

1. Live local AI summaries through Ollama.
2. Telegram alerts with a clickable public dashboard link.
3. A public Cloudflare-powered mobile dashboard.
4. Windows runtime resilience through Docker health checks, startup automation, and watchdog repair scripts.

It also adds Zafronix as the preferred live World Cup 2026 data provider after API-Football free tier access was confirmed to block 2026 fixture data.

## Key Outcomes

- The dashboard is publicly reachable through the configured Cloudflare hostname.
- Telegram test alerts are working and include the dashboard link.
- The backend can reach the Windows-hosted Ollama service through `host.docker.internal`.
- The AI health endpoint reports the local Llama model as available.
- Provider sync uses Zafronix and successfully imports real World Cup 2026 fixtures.
- Docker Compose services include health checks for backend, dashboard, and Postgres.
- Windows scheduled tasks can start and watch the runtime automatically.
- The watchdog can repair stopped backend/dashboard containers.
- Cloudflared runs as a Windows service using the correct tunnel configuration.

## Live Provider Decision

API-Football remains supported, but its free plan blocks 2026 fixture data.

Because of that limitation, Zafronix was added as the preferred provider for this project.

The selected runtime provider is controlled by:

```env
FOOTBALL_API_PROVIDER=zafronix
ZAFRONIX_BASE_URL=https://api.zafronix.com/fifa/worldcup/v1
ZAFRONIX_API_KEY=replace_me
ZAFRONIX_WORLD_CUP_YEAR=2026
```

## Why 72 Fixtures Instead of 104?

Zafronix returned 104 raw World Cup 2026 match rows during manual API testing.

The app imports only rows with usable fixture data, including real home team, away team, and kickoff time. Knockout fixtures with placeholder teams are skipped for now to avoid polluting the dashboard with incomplete data.

This keeps the current dashboard reliable for group-stage and confirmed fixtures.

## Windows Runtime Resilience

Windows helper scripts under `scripts/windows/` support:

- Starting Docker Desktop if needed.
- Starting the Docker Compose runtime.
- Checking backend, dashboard, public health, and AI health.
- Restarting unhealthy containers.
- Ensuring the Cloudflare tunnel service is running.
- Running hidden scheduled tasks without flashing PowerShell windows.

## Final Notes

This release is suitable for personal use during the World Cup:

- Generate local AI summaries.
